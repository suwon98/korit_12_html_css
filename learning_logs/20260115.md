## 갖다 놓는(배치하는) HTML / 꾸며주는 CSS / 동작시키는 JS
- 저희가 Java를 배우면서 작성한 것은 code라는 표현을 썼지만, HTML + CSS로
작업한 것은 문서라고 합니다. 즉, 얘네 둘을 이용해서 만든 것은 프로그램이 아니라
document에 해당합니다.

- 웹 페이지에 들어갈 요소들을 배치하고 디자인하는 정도인 .html 파일과 .css파일은
브라우저라는 프로그램(저희는 크롬 사용합니다)에서 불러와서 보여주는 문서의 의미입니다.

- 하지만 JS의 경우 서버로부터 전달 받아 브라우저에서 동작하는 파일에 해당합니다. 하지만
Node.js 등 컴퓨터에 설치된 JS 엔진에서 python / java와 같은 언어처럼 복잡한
알고리즘을 정의하고 컴퓨터에 명령을 내릴 수 있습니다.

이상을 이유로 JS는 프로그래밍 언어이기 때문에 code라고 하고, HTML + CSS는 문서라고 합니다.

## 웹 디자이너 vs. 웹 퍼블리셔
웹디자이너가 작업한 결과물을 HTML + CSS 문서로 작성하는 사람을 우리나라에서는 웹 퍼블리셔라는 표현을 씁니다.
여기에 JS까지 적용하여 기능을 만들어 낼 수 있는 사람을 웹 개발자라고 할 수 있겠습니다.

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>docoudment</title>
</head>
<body>
  <h1></h1>

</body>
</html>
```
`! + tab`으로 자동 생성된 것은 이상과 같습니다. 이는 웹 개발 시에 기계 또는
시스템에 전달하는 각종 정보로, _HTML 문서 작성 시에 꼭 필요한 내용_ 들을 담고 있습니다.
문서 제목, 작성자, 사용 언어 및 각종 메타 정보(메타 데이터) 등으로 구성되어 있는데,
어차피 형식 자체는 동일하기 때문에 일일이 타이핑할 필요 없이 단축키를 지원합니다. 이를
Emmet이라고 하는데 `! + tab`도 이 중 하나에 속합니다. 앞으로 에밋이라고 했을 떄,
특정 ㅜㄴ서 작성을 바르게 해주는 단축키로 생각하시면 되겠습니다.
그리고 `! + tab`은 저희가 .html파일 생성할 때 마다 맨 처음에 하는 작업이 될 겁니다.


- `<html lang="en">`의 의미
lang 속성은 HTML 문서의 웹사이트가 어떤 언어로 되어있는지 _컴퓨터에게 알려주는_
역할을 합니다. 이걸 지정해줬을 때 구글 / 네이버 등에서 검색할
때 한글로 된 웹사이트를 찾는 것이 가능합니다. 또한 시각 장애인들을 위한
컴퓨터가 웹 사이트를 분석할 때도 쓰입니다. 그리고 맥 사용자의 경우
lang="en"인데 캡스락 눌러서 한글로 웹 페이지 작성하면 기본 서체가 궁서체 입니다.

## 첫 태그 사용하기
`<어쩌고>` + `</어쩌고>` : `/`가 있는 쪽은 닫는 태그에 해당함

```html
<div>
  <p>
    안녕하세요.
  </p>
</div>
```
라고 가정했을 때 div 태그 내에 p태그가 종속되어있다고 볼 수 있습니다.
이를 확인할 수 있도록 들여쓰기(indent-writing)를 사용하여 다른
개발자들도 쉽게 알아채도록 합니다.

근데 가끔 닫는 태그가 없는 태그도 존재합니다. `<br>`
MDN 공식 문서(HTML 관련) 상에서는 `<br>`을 `<br/>`로 쓰지말 것을
권유하고 있습니다.

근데 리액트에서는 `<br>`로 쓰면 오류나서 `<br/>`로 써야 합니다.

1. `<hr>` : horizontal의 축약어로 가로줄을 의미함.
<hr>

<h1>안녕하세요<h1/>
<div>여기는 분할되는 부분입니다
  <p>그리고 또 안에 들어가는 부분도 쓸 수 있구요
  </p>
</div>
